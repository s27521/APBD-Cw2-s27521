# APBD-Cw2-s27521
Próby zadbania o kohezję i coupuling widoczne jest przy podziale różnych klas na różne katalogi.
Serwisy uzyskują informację od modeli na podstawie gotowych metod zwracających te informacje, 
dzięki czemu problemy po stronie serwisowej nie będą potrzebowały modyfikować modeli do rozwiązywania problemów.
Logiki biznesowe są umieszczane w odpowiednich klasach uwzględniając odpowiedzialności
Wybrałem ten model, gdyż wydaje on się dla mnie najbardziej optymalny do podanego zadania.
